# IPC Audit Report - May 16, 2026

## Overview
This report documents the findings of an IPC (Inter-Process Communication) integrity audit conducted on the SuperFokus codebase. The audit compared handlers in the Main process against invocations and listeners in the Renderer process.

## ⚠️ Critical Discrepancies

### 1. Missing Main Handlers (Silent Failures)
These channels are invoked by the Renderer but have no corresponding `ipcMain` handler in the Main process.

- **`blocker-expand-fullscreen`**
    - **Source:** `src/renderer/ui/popup.js:33`
    - **Impact:** High. The site blocker's "lock" functionality is broken; the countdown finishes, but the window never expands to fullscreen.
    - **Fix:** Add a handler in `BlockerService.js` or `IpcMainHandlers.js`.

- **`blocker-show-popup`**
    - **Source:** Whitelisted in `src/main/preload.js:24`
    - **Impact:** Low (Potential missing feature). No code currently sends this, and no handler exists.
    - **Fix:** Either implement the intended feature or remove it from the whitelist.

### 2. Missing Renderer Listeners
These channels are sent from the Main process but are not being listened for in the Renderer.

- **`startup-cleanup-failed`**
    - **Source:** `src/main/main.js:58`
    - **Impact:** Medium. If the app fails to clean up old hosts file entries on startup, the user is not notified.
    - **Fix:** Add an `ipcRenderer.on` listener in the dashboard UI.

## 🔍 Code Health Observations

### Unused Handlers (Dead Code)
- **`store-delete`**
    - **Handler:** `src/main/services/IpcMainHandlers.js:28`
    - **Status:** Functional but uncalled. 
    - **Recommendation:** Keep for future-proofing or remove if unused in the next major version.

### Verified Alignments (Working Correctlry)
- All Timer Services (`start-timer`, `stop-timer`, `pause-timer`, `resume-timer`) are perfectly synced.
- All Pomo, Flow, and Micro-Sprint specific UI controls are correctly mapped.
- Dynamic prefix-based timer channels (`timer-tick`, `timer-complete-*`) are correctly whitelisted and handled.

## Conclusion
The core of the IPC bridge is stable, but the **Site Blocker's fullscreen expansion** is currently non-functional due to a missing handler. This should be the priority for the next fix.
